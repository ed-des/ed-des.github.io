html layout essentials that jia learned from this exercise

task 1: create group of elements contained in a parent element, and attribute a style to the parent element.

why would you want one element put inside another element?

so that you can group elements together, and then design that group in a certain way.

task 2: create a layout comprised of a sequence of inline elements, one after the other.

when you add inline elements on a page they will be laid out left-to-right top-to-bottom

task 3: create a layout comprised of a sequence of block elements, one going after the other.

block elements stack top-to-bottom
each element stretches horizontally to take up its own entire line
other inline or block context cannot appear to the left or right of the block element

task 4: create a layout of elements, where at least one inline element was turned into a block element, vice versa.

i am a div, which displays as block by default, but am displaying inline instead
i am a span, which display as inline by default, but am displaying block instead

task 5: create a layout comprised of inline-block elements.

an inline-block element will be built as if it were a block element
but it will flow on the page as if it were an inline element

task 6: create a layout comprised inline content that is centered in its container.

the parent div has the style attribute of text-align: center; which centers both inline span elements in the parent.

task 7: create a layout comprised a block element that is centered in its container.

the div is both centering itself in its own parent and it is centering all inline text inside it.

task 8: create a layout comprised an element floating amid other inline elements.

float aligns block and inline elements and it will overlap with sibling block elements but not inline elements float
